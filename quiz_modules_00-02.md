## Вопросы

### Вопрос 1
**Что такое «кластер баз данных» в терминологии PostgreSQL?**

A) Группа серверов PostgreSQL, работающих в режиме репликации
B) Набор баз данных, управляемых одним экземпляром PostgreSQL
C) Объединение нескольких серверов для балансировки нагрузки
D) Резервная копия всех баз данных сервера

<details>
<summary><b>Ответ: B</b></summary>
Кластер баз данных — это набор баз данных, с которыми одновременно работает один экземпляр PostgreSQL. Это данные в файлах, которыми управляет сервер.
</details>

---

### Вопрос 2
**Какая утилита используется для инициализации кластера баз данных?**

A) pg_ctl
B) pg_createcluster
C) initdb
D) psql

<details>
<summary><b>Ответ: C</b></summary>
initdb — основная утилита для инициализации кластера. pg_createcluster — это обёртка над initdb, специфичная для Ubuntu.
</details>

---

### Вопрос 3
**Какой ключ утилиты initdb включает подсчёт контрольных сумм страниц?**

A) -c
B) -k
C) --checksums
D) -s

<details>
<summary><b>Ответ: B</b></summary>
Ключ -k включает подсчёт контрольных сумм страниц данных, что позволяет своевременно обнаруживать повреждения данных.
</details>

### Вопрос 4

### Вопрос 7
**Включён ли по умолчанию расчёт контрольных сумм при создании кластера через пакет в Ubuntu?**

A) Да, включён
B) Нет, не включён
C) Зависит от версии PostgreSQL
D) Зависит от версии Ubuntu

### Вопрос 8
**Какой каталог в кластере содержит журнал упреждающей записи (WAL)?**

A) pg_xact
B) pg_wal
C) pg_log
D) base

<details>
<summary><b>Ответ: B</b></summary>
Каталог pg_wal содержит журнал упреждающей записи (Write-Ahead Log). До PostgreSQL 10 он назывался pg_xlog.
</details>

### Вопрос 9
**Что делает команда `pg_ctlcluster 16 main reload`?**

A) Перезапускает сервер
B) Останавливает и запускает сервер
C) Перечитывает файлы конфигурации без перезапуска
D) Переключает на реплику

<details>
<summary><b>Ответ: C</b></summary>
Команда reload перечитывает файлы конфигурации (postgresql.conf, pg_hba.conf) без перезапуска сервера. В отличие от restart, не прерывает соединения клиентов.
</details>

### Вопрос 10
**Какой файл содержит информацию о запущенном сервере (PID, параметры запуска)?**

A) PG_VERSION
B) postgresql.auto.conf
C) postmaster.pid
D) postmaster.opts

<details>
<summary><b>Ответ: C</b></summary>
Файл postmaster.pid содержит PID сервера и параметры его запуска. Он создаётся при запуске и удаляется при останове.

### Вопрос 14
**Что означает символ `*` в приглашении psql (`=*#`)?**

A) Подключён суперпользователь
B) Активна транзакция
C) Произошла ошибка
D) Режим автоматической фиксации

<details>
<summary><b>Ответ: B</b></summary>
Символ `*` в приглашении означает, что транзакция активна (выполнена команда BEGIN). Символ `!` означает, что транзакция прервана ошибкой.
</details>

---

### Вопрос 15
**Что происходит в psql по умолчанию при ошибке внутри открытой транзакции?**

A) Откатывается только ошибочная команда
B) Вся транзакция прерывается и требует завершения
C) psql автоматически выполняет ROLLBACK
D) Ошибка игнорируется

<details>
<summary><b>Ответ: B</b></summary>
PostgreSQL не умеет откатывать только одну команду транзакции. После ошибки вся транзакция прерывается — все последующие команды игнорируются до COMMIT или ROLLBACK.
</details>

---

### Вопрос 16
**Что делает переменная ON_ERROR_ROLLBACK = on в psql?**

A) Автоматически выполняет ROLLBACK при любой ошибке
B) Устанавливает SAVEPOINT перед каждой командой в транзакции
C) Останавливает выполнение скрипта
D) Отключает AUTOCOMMIT

<details>
<summary><b>Ответ: B</b></summary>
ON_ERROR_ROLLBACK = on заставляет psql устанавливать SAVEPOINT перед каждой командой SQL в открытой транзакции. При ошибке происходит откат к этой точке сохранения, и можно продолжать работу.
</details>

### Вопрос 20
**Для чего используется утилита pg_checksums?**

A) Для проверки целостности WAL
B) Для включения/отключения/проверки контрольных сумм страниц данных
C) Для проверки контрольных сумм в индексах
D) Для вычисления хешей таблиц

<details>
<summary><b>Ответ: B</b></summary>
pg_checksums позволяет включать (--enable), отключать (--disable) и проверять (--check) контрольные суммы страниц данных. Сервер должен быть остановлен для включения/отключения.
</details>

### Вопрос 25
**Как настроить приглашение psql в формате `пользователь@база=#`?**

A) `\set PROMPT1 '%n@%/%R%x%# '`
B) `\set PROMPT '%u@%d=# '`
C) `\setenv PROMPT1 '%n@%/# '`
D) `\prompt '%n@%/%R%x%# '`

<details>
<summary><b>Ответ: A</b></summary>
Переменные PROMPT1 и PROMPT2 определяют вид приглашения. Спецификаторы: %n — имя пользователя, %/ — имя БД, %R — статус транзакции, %x — статус транзакции для COPY, %# — символ привилегий (# для суперпользователя, > для обычного).
</details>
