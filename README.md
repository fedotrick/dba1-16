# 🐘 PostgreSQL DBA1 Master — Интерактивный Тренажер

Интерактивное веб-приложение для подготовки к сертификации **«Администрирование PostgreSQL 16. Часть 1» (DBA1-16)**. 

Этот проект объединяет теоретические конспекты, тренировочные тесты по каждому модулю и полноценную симуляцию финального экзамена.

---

## 🚀 Основные возможности

- **19 Учебных Модулей:** Полный охват программы курса DBA1-16, включая архитектуру (MVCC, WAL, VACUUM), мониторинг и репликацию.
- **Интерактивные Квизы:** Проверка знаний сразу после изучения теории с мгновенным результатом и подробными объяснениями.
- **Спецвыпуск PG16:** Отдельный блок по нововведениям PostgreSQL 16 (включая `pg_stat_io` и новые роли).
- **Режим Экзамена:** Симуляция реального теста на 50 вопросов с таймером (75 минут) и проходным баллом 75%.
- **Сохранение прогресса:** Информация о пройденных модулях хранится локально в браузере.
- **Docker-ready:** Готовая конфигурация для развертывания на собственном сервере.

---

## 📂 Структура курса

1. **Инструментарий:** Установка, `psql`, конфигурирование.
2. **Архитектура:** Общее устройство, MVCC, VACUUM, WAL.
3. **Данные:** Базы данных, системный каталог, Tablespaces, низкий уровень хранения.
4. **Администрирование:** Мониторинг, управление доступом, бэкапы.
5. **Репликация:** Обзор физической и логической репликации.
6. **PRO-блоки:** Особенности версии 16 и глубокая шпаргалка по `pg_catalog`.

---

## 🛠️ Технологический стек

- **Frontend:** React + Vite + TypeScript + Tailwind CSS
- **Data Parsing:** Python 3 (автоматическая конвертация Markdown-заметок в JSON)
- **Deployment:** Docker + Nginx

## 💻 Локальный запуск (Development)

Если вы хотите запустить проект на своем компьютере для разработки:

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://gitverse.ru/warriorpacis/dba1-16.git
   cd dba1-16
   ```

2. **Подготовьте данные:**
   ```bash
   python parse_data.py
   ```

3. **Установите зависимости и запустите сервер:**
   ```bash
   npm install
   npm run dev
   ```
   Сайт будет доступен по адресу `http://localhost:3000`.

---

## 🐳 Запуск через Docker (Production)

Для развертывания на сервере или в домашней сети:

```bash
docker-compose up -d --build
```
Проект будет доступен на порту `8080`.
