## Модуль 01: Установка и управление сервером

### Основные понятия

**PostgreSQL** — программа класса СУБД (система управления базами данных).

Когда эта программа выполняется, мы называем её:
- **сервер PostgreSQL** или **экземпляр сервера**

**Кластер баз данных** — набор баз данных, с которыми одновременно работает один экземпляр PostgreSQL.

> **Кластер баз данных** = данные в файлах
> **Сервер/экземпляр** = программа, управляющая кластером

### Установка из исходных кодов

**Зачем:**
- Можно использовать нестандартные параметры
- Можно собрать на специфичной архитектуре
- Стабильная версия: https://www.postgresql.org/ftp/source (gzip, bzip2)

**Из git-репозитория:**
- `git://git.postgresql.org/git/postgresql.git`
- Текущая версия на любой коммит
- Требуется более широкий набор инструментов (Flex, Bison и др.)
- Необходимо в первую очередь для разработчиков ядра

### Необходимое ПО для сборки

**Обязательно:**
- `tar`, `gzip/bzip2`
- `GNU make`
- Компилятор Си (C89)

**Используются, но можно отключить:**
- GNU Readline, zlib, lz4, zstd

**Дополнительно:**
- Perl, Python, Tcl — для PL/Perl, PL/Python, PL/Tcl
- Kerberos, OpenSSL, OpenLDAP, PAM — для аутентификации и шифрования
- ICU — для кроссплатформенной поддержки UNICODE

### Сборка и установка (традиционный способ)

```bash
# Распаковка
student$ tar xzf postgresql-16.0.tar.gz
student$ cd postgresql-16.0

# Конфигурация
student$ ./configure --prefix=/home/student/pgsql16

# Сборка
student$ make

# Установка
student$ make install
```

**Начиная с PostgreSQL 16** — также можно использовать систему сборки **Meson**:
```bash
meson setup build
ninja -C build
ninja -C build install
```

### Инициализация кластера баз данных

**Утилита `initdb`:**

```bash
# Установка переменных окружения
student$ export PGDATA=/home/student/pgsql16/data
student$ export PATH=/home/student/pgsql16/bin:$PATH

# Инициализация
student$ initdb -U postgres -k -D /home/student/pgsql16/data
```

**Ключи initdb:**
| Ключ | Значение |
|------|----------|
| `-U имя` | Имя суперпользователя СУБД (по умолчанию — имя пользователя ОС) |
| `-k` | Включить подсчёт контрольных сумм страниц |
| `-D каталог` | Каталог для данных (если не задана PGDATA) |

### Управление сервером — утилита `pg_ctl`

**Основные задачи:**
- Запуск, останов и получение статуса сервера
- Обновление параметров конфигурации
- Переключение на реплику

```bash
# Запуск сервера
student$ pg_ctl start -l /home/student/logfile

# Проверка работы
student$ psql -U postgres -p 5555 -c 'SELECT now();'

# Останов сервера
student$ pg_ctl stop
```

Запускать `pg_ctl` следует от имени **владельца кластера баз данных**.

### Пакетная установка (предпочтительный способ)

**Linux:** входит в дистрибутив или репозиторий (yum, apt, RPM, DEB)
**FreeBSD/OpenBSD:** пакеты из Ports and Packages Collection
**Mac OS X, Windows:** готовые пакеты

Репозиторий PGDG: https://www.postgresql.org/download/

### Установка в Ubuntu

```bash
student$ sudo apt install -y postgresql-16
```

**Каталог установки:** `/usr/lib/postgresql/16/`
- `bin/` — исполняемые файлы
- `lib/` — библиотеки

**Кластер создаётся автоматически** с именем `main`:
- Расположение: `/var/lib/postgresql/16/main/`
- Владелец: `postgres`

### Утилита `pg_createcluster` (Ubuntu)

Обёртка над `initdb`, специфичная для Ubuntu:
- Инициализация выполняется при установке пакета
- Создаёт кластер «main»
- Контрольные суммы **по умолчанию НЕ включены**
- Автоматический запуск сервера при старте ОС

```bash
# Создание кластера
pg_createcluster 16 mycluster

# Удаление кластера
pg_dropcluster 16 mycluster
```

### Управление сервером в Ubuntu — `pg_ctlcluster`

В пакетном дистрибутиве Ubuntu доступ к `pg_ctl` через обёртку `pg_ctlcluster`:

```bash
# Остановить сервер
student$ sudo pg_ctlcluster 16 main stop

# Запустить сервер
student$ sudo pg_ctlcluster 16 main start

# Перезапустить
student$ sudo pg_ctlcluster 16 main restart

# Получить статус
student$ sudo pg_ctlcluster 16 main status

# Перечитать конфигурацию
student$ sudo pg_ctlcluster 16 main reload
```

**`pg_ctlcluster` принимает:**
1. Номер версии сервера (16)
2. Название сервера (main)
3. Команду (start, stop, restart, reload, status)

### Утилита `pg_lsclusters`

```bash
student$ pg_lsclusters
Ver  Cluster  Port  Status  Owner   Data directory                    Log file
16   main     5432  online  postgres /var/lib/postgresql/16/main     /var/log/postgresql/postgresql-16-main.log
```

### Расположение файлов при пакетной установке (Ubuntu)

| Что | Где |
|-----|-----|
| Данные кластера | `/var/lib/postgresql/16/main/` |
| Файлы конфигурации | `/etc/postgresql/16/main/` |
| Журнал сервера | `/var/log/postgresql/postgresql-16-main.log` |

### Структура каталога данных кластера

```
base/          — данные пользовательских БД
global/        — глобальные таблицы (роли, БД, табличные пространства)
pg_commit_ts/  — данные о времени фиксации транзакций
pg_dynshmem/   — разделяемая память
pg_logical/    — данные логического декодирования
pg_multixact/  — данные для групповых транзакций
pg_notify/     — данные LISTEN/NOTIFY
pg_replslot/   — слоты репликации
pg_serial/     — данные о сериализуемых транзакциях
pg_snapshots/  — экспортированные снимки
pg_stat/       — статистика
pg_stat_tmp/   — временная статистика
pg_subtrans/   — данные о подтранзакциях
pg_tblspc/     — ссылки на табличные пространства
pg_twophase/   — файлы подготовленных транзакций
pg_wal/        — журнал упреждающей записи (WAL)
pg_xact/       — данные о состоянии транзакций
PG_VERSION     — версия PostgreSQL
postgresql.auto.conf — авто-конфигурация
postmaster.opts — параметры запуска
postmaster.pid — PID-файл
```

### Контрольные суммы — утилита `pg_checksums`

```bash
# Проверка
sudo /usr/lib/postgresql/16/bin/pg_checksums --check -D /var/lib/postgresql/16/main

# Включение (сервер должен быть ОСТАНОВЛЕН)
sudo /usr/lib/postgresql/16/bin/pg_checksums --enable -D /var/lib/postgresql/16/main
```

### Облачные решения

**PostgreSQL в виртуальной среде:**
- Накладные расходы на виртуализацию
- Администратор имеет полный доступ к экземпляру

**PostgreSQL as a Service (DBaaS):**
- Основное администрирование берёт на себя провайдер
- Доступ только к ограниченным инструментам (мониторинг, бэкапы)
- Нет полного доступа к экземпляру

2. Доступны облачные решения
3. Сервер выполняется под выделенным пользователем ОС
4. Перед использованием сервера необходимо инициализировать кластер баз данных
5. Специальные команды для управления сервером

## Модуль 02: Использование psql

### Назначение
- Терминальный клиент для работы с PostgreSQL
- Поставляется вместе с СУБД
- Используется администраторами и разработчиками
- Для интерактивной работы и выполнения скриптов

Дока: https://postgrespro.ru/docs/postgresql/16/app-psql

### Подключение к серверу

**Запуск из командной строки:**
```bash
$ psql -d база -U пользователь -h узел -p порт
```

**Параметры по умолчанию:**
| Параметр | Значение по умолчанию |
|----------|----------------------|
| база | совпадает с именем пользователя |
| пользователь | совпадает с именем пользователя ОС |
| узел | соединение через Unix socket |
| порт | обычно 5432 |

**Новое подключение из psql:**
```sql
\c[onnect] [база] [пользователь] [узел] [порт]
```

**Информация о текущем подключении:**
```sql
\conninfo
```

### Получение справки

**В командной строке ОС:**
```bash
$ psql --help
$ man psql
```

**В psql:**
| Команда | Описание |
|---------|----------|
| `\?` | Список команд psql |
| `\? variables` | Переменные psql |
| `\h[elp]` | Список команд SQL |
| `\h команда` | Синтаксис команды SQL |
| `\q` | Выход из psql (также `exit`, `quit`) |

> Все команды psql начинаются с обратной косой черты `\` и, как правило, их можно сокращать до первой буквы.

### Форматирование результатов

| Команда | Описание |
|---------|----------|
| `\pset format aligned` | Выровненный формат (по умолчанию) |
| `\pset format unaligned` | Невыровненный формат |
| `\pset format html` | HTML-таблица |
| `\pset format csv` | CSV-формат |
| `\pset format json` | JSON |
| `\pset border 2` | Рамки вокруг ячеек и таблицы |
| `\x auto` | Автоматический развёрнутый вывод |
| `\x on` | Всегда развёрнутый вывод |
| `\pset null 'null'` | Отображение NULL |
| `\pset pager always` | Всегда использовать постраничный просмотр |

### Вывод дополнительной информации

| Команда | Описание |
|---------|----------|
| `\timing on` | Показывать время выполнения команд |
| `\set PROMPT1 '...'` | Настройка приглашения |

### Работа с транзакциями

**По умолчанию:** AUTOCOMMIT = on — каждая команда в отдельной транзакции.

**Приглашение psql показывает статус транзакции:**
| Символ | Значение |
|--------|----------|
| `=#` | Нет активной транзакции (суперпользователь) |
| `=>` | Нет активной транзакции (обычный пользователь) |
| `=*#` | Транзакция активна |
| `=!#` | Транзакция прервана ошибкой |

> **Важно:** PostgreSQL не умеет откатывать только одну команду транзакции. После ошибки вся транзакция прерывается и её нужно завершить (COMMIT или ROLLBACK).

### Переменная ON_ERROR_ROLLBACK

```sql
\set ON_ERROR_ROLLBACK on
```

Заставляет psql устанавливать **SAVEPOINT** перед каждой командой SQL в открытой транзакции. В случае ошибки — автоматический откат к точке сохранения, можно продолжать работу.

Значение `interactive` — такое поведение только в интерактивном режиме, но не при выполнении скриптов.

### Команды для работы с файлами

| Команда | Описание |
|---------|----------|
| `\i файл` | Выполнить команды из файла |
| `\o файл` | Записать вывод в файл |
| `\o` (без аргументов) | Вернуть вывод на экран |
| `\e` | Открыть редактор для ввода запроса |
| `\ef функция` | Редактировать функцию |

### Переменные psql

```sql
-- Установка переменной
\set имя значение
\set top5 'SELECT * FROM pg_tables LIMIT 5;'

-- Использование
:имя
:top5

-- Удаление переменной
\unset имя

-- Показать все переменные
\set
```

**Системные переменные (автоматически устанавливаются):**

| Переменная | Описание |
|------------|----------|
| `DBNAME` | Текущая БД |
| `USER` | Текущий пользователь |
| `HOST` | Хост подключения |
| `PORT` | Порт подключения |
| `VERSION` | Версия PostgreSQL |
| `SERVER_VERSION_NAME` | Полное имя версии сервера |
| `SERVER_VERSION_NUM` | Номер версии сервера |
| `AUTOCOMMIT` | Автоматическая фиксация |
| `ECHO` | Отображение команд |
| `ON_ERROR_STOP` | Остановка при ошибке в скрипте |
| `ON_ERROR_ROLLBACK` | Откат к savepoint при ошибке |
| `HISTFILE` | Файл истории |
| `HISTSIZE` | Размер истории |
| `PROMPT1` | Приглашение для первой строки |
| `PROMPT2` | Приглашение для последующих строк |
| `PROMPT3` | Приглашение для команды COPY |

### Переменные и условные операторы

```sql
-- Проверка существования переменной
\echo :{?имя_переменной}

-- Условный оператор
\if :{?working_dir}
    -- переменная определена
\else
    \set working_dir `pwd`
\endif
```

### Настройка psql — файлы конфигурации

При запуске выполняются два скрипта (при их наличии):
1. **Системный** `psqlrc` — расположение: `pg_config --sysconfdir`
2. **Пользовательский** `~/.psqlrc`

```bash
# Узнать расположение системного psqlrc
student$ pg_config --sysconfdir
```

**Что можно настроить в `.psqlrc`:**
- Приглашение psql (`\set PROMPT1`)
- Программу постраничного просмотра (`\setenv PSQL_PAGER 'less -XS'`)
- Переменные для часто используемых команд (`\set top5 '...'`)
- Время выполнения команд (`\timing on`)
- Автодополнение и история (readline)

### Команды для работы с системным каталогом

Серия команд, начинающихся на `\d`:

| Команда | Описание |
|---------|----------|

| `\di` | Список индексов |

| `\d имя` | Описание таблицы/представления/индекса |
| `\d+ имя` | Расширенное описание с размером и описанием |
| `\dn` | Список схем |
| `\df` | Список функций |
| `\l` | Список баз данных |
| `\l+` | Подробный список баз данных |
| `\du` | Список ролей |
| `\du+` | Подробный список ролей |
