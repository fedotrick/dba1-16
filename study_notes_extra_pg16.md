# 📝 Конспект: Дополнения по PostgreSQL 16 и Системный каталог

---

## Модуль 17: Особенности PostgreSQL 16

### 1. Новое в мониторинге: pg_stat_io
Это одно из самых важных изменений. Представление `pg_stat_io` позволяет детально видеть статистику ввода-вывода (I/O) для различных типов объектов и процессов.
- **Зачем:** Раньше было трудно понять, кто именно "грузит" диск — чекпоинтер, фоновая запись или обычные бекенды.
- **Колонки:** `backend_type`, `object`, `context`, `reads`, `writes`, `extends`.

### 2. Безопасность и Роли
В 16-й версии расширена концепция **Predefined Roles** (предопределенных ролей). Это позволяет давать права без `SUPERUSER`.
- `pg_read_all_data` / `pg_write_all_data`: Чтение/запись всех данных во всех БД.
- `pg_monitor`: Набор прав для инструментов мониторинга.
- `pg_checkpoint`: Право выполнять команду `CHECKPOINT`.
- **Новое:** Теперь можно давать право на `GRANT` определенных ролей другим пользователям без участия суперпользователя (через `ADMIN OPTION`).

### 3. Логическая репликация
- **Репликация со стендбая:** Теперь подписчик может подключаться к физическому стендбаю (slave), а не только к мастеру. Это снижает нагрузку на основной сервер.
- **Параллельное применение:** Подписчик может применять транзакции параллельно (параметр `streaming = parallel`).

## Модуль 18: Шпаргалка по Системному каталогу (pg_catalog)

На экзамене часто просят найти метаданные через SQL. Вот главные таблицы:

| Таблица/Представление | Что там лежит |
|-----------------------|---------------|
| **pg_class** | Все объекты, имеющие колонки (таблицы, индексы, представления). Поле `relkind`: `r`=table, `i`=index, `v`=view. |
| **pg_attribute** | Колонки таблиц. Связана с `pg_class` через `attrelid`. |
| **pg_type** | Типы данных. |
| **pg_index** | Детальная информация об индексах. |
| **pg_database** | Список всех баз данных в кластере. |
| **pg_tablespace** | Табличные пространства. |
| **pg_settings** | Текущие настройки сервера (аналог `SHOW ALL`). |

### Полезные функции для экзамена:
- `pg_column_size(any)` — размер конкретного значения.
- `pg_relation_size('table_name')` — размер только данных таблицы (без индексов).
- `pg_total_relation_size('table_name')` — полный размер таблицы со всеми индексами и TOAST.
- `pg_reload_conf()` — перечитать конфигурацию.
